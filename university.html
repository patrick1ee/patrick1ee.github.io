Applied Cryptography Project

An individual project undertaken during the latter term of my third year, aiming to explore both power analysis attacks on AES-128 implementations and respective countermeasures, as well as consider broader challenges within cryptographic engineering.

Achievements:

  • Implemented a power analysis attack in Python based on the Correlation Power Analysis attack described by Brier et al. and making use of the Pico oscilloscope. This was successful in recovering the entire encryption key from an AES implementation running on the SCALE micro-controller.
  • Dramatically increased the speed of this attack by using einstein summations to correlate matrices, taking the analysis time down from several minutes to several seconds meaning the entire attack, including trace acquisition, took ~ 2 minutes
  • Implemented a successful countermeasure to this attack in C based on both randomly disaligning the order of operation on the encryption state and additive masking, with the encryption functionality running within an efficiency budget of 100ms
  • Analysed the performance and efficiency of each component of the countermeasure as well as the trade-off between efficiency and the memory constraints of the device
  • Wrote a specification analysing possible threats, countermeasures and implementation techniques were the implementation to be extended to support functionality required for TLS communication.

Druid Drum Circle (Video Game Project)

A group project undertaken during the latter term of my third year, developing an immersive rhythm game from scratch using Unity. This was an intense teamwork experience, requring an AGILE development methodology and effective use of git for version control.

Group Achievements:

  • Designed and built our own drum MIDI controllers using pizeo sensors, an arduino controller and LED lights, using a laser cutter to create the wodden components of the drum container.
  • Implemented the ability for the game to read in music from audio files and parse it into a series of timestamped notes
  • Replaced the existing Unity Kinect library with our own version, allowing for a single Kinect camera to track multiple bodies seperately.
  • Implemented the ability for a variety of trees to grow completely procedurally, with each one being completely unique in both appearance and its growth animation. This was based on research by Jinmo Kim with modifications to fit our needs
  • Designed 3 immersive environments using a combination of unity's built in terrain builder and detailed modelling for more granular elements
  • Developed our game so that it could be played in a 270 degree immersive tent, using three screens to give the illusion of virtual reality. This required further optimisations as the game would effectively have to produce three different renders on every frame.

Individual focuses:

  • Built upon work by McFee et al. to create functionality for analysing audio files, using spectral flux analysis to create a map of beat onsets against time, including a differentiator to determine if the beat was part of the underlying rhythm (e.g. a kick drum) or more of an off-beat transient (e.g. a hi-hat or tom) and store the resulting data as a JSON array.
  • Implementing a MIDI interface so that our custom drum controllers could be used with the game
  • Developing the UI, particularly the in-game rhythm markers which would appear in time with the music
  • Integrating major features, such as the procedural trees into the bass version of the game and

Druid Drum Circle is a rhythm game in-which players must drum in time with the music, and the corresponding rhythm markers, in order to score points and cause foliage to grow in an otherwise treeless environment. Players can pick between a 'forest', 'mountains', and 'beach' environment, which their druid avatars, reflecting the movements of the players from the Kinect, will fly around on a platform. At the end of the level, the camers will pan around the environment, showing the lush foliage which has now appeared and the player with the highest score wins.

Machine Learning Project

An individual project undertaken during the first term of my third year, requring in-depth use of a variet of machine learning techniques including Mixture Models, Neural Networks, Support Vector Machines, Bayesian Regression, Trees and Ensembles.

Achievements:

  • Fit a Gaussian Mixture model to the public fashion-MNIST dataset and analysed the classification accuracy by using Completeness and Homogeneity scores
  • Fit a feed-forward Neueral Network to the fashion-MNIST dataset, achieving a classifcation accuracy of 86% and whilst analysing the trade-off between number of hidden neurons and training time.
  • Used a randomized search to find optimal hyper-parameters for this nerual network and assessed which hyper-parameters had the largest overall effect on performance
  • Fit a support vector machine to this dataset and performed hyper-parameter tuning and analysis, similar to with the neural network
  • Used Bayesian linear regression to predict median house value from the California Housing dataset, using the Hamiltonian Markov Chain Monte Carlo to obtain posterior distributions for each model parameter, and assessing performance in comparison to least squares regression and when fitted to different sample sizes.
  • Used a grid search to tune the hyper-parameters of a CART decision tree for use with the California Housing dataset, analysing each parameters effect on model performance
  • Analysed the difference in accuracy of the tuned CART decision tree with that of Bayesian linear regression for the same dataset
  • Analysed the tradeoff of number of estimators with training time for a Random Forest Regressor
